Discovery Patterns

Turning strings into things

  • Usually the first step in an application
  • Not just limited to strings.

Reconciliation Pattern

Problem

The user wants to specify an exact entity rather than just keywords.

Keywords can be ambiguous so we need to be able to present them with options.

Not just limited to keywords. Images, articles, geo-coordinates, etc. can all be reconciled to entities.

Reconciliation Pattern

Solution

Present the user with a list of disambiguated entities.

Use reconciliation algorithms to automatically match entities.

Combine both techniques to get the best results.
source: Ranker
  • Ranker.com is a site that lets you create and share ranked lists of things.
  • Here we see their list editor which prompts you to enter a new entity.
source: Ranker
  • The autocomplete box looks up possible matching entities in their database.
  • Lets the user create a new entity if needed.

Detection Pattern

Problem

User wants to learn more about the subject matter of an article, video, image or other piece of content.

Detection Pattern

Solution

Use Named Entity Recognition.

Parse structured markup like Schema.org, RDFa and Microformats.
source: DBpedia Spotlight
  • DBpedia Spotlight is an open source project.
  • DBpedia is a project to extract structured data about entities from Wikipedia.
  • Named entity recognition using DBpedia data
  • DBpedia IDs can be easily mapped to other sources like Freebase.

Extraction Pattern

When to use it?

Lots of existing content without entities identified.

Need to identify relevant entities.

Too much content to manually mark it up with entities.
source: OpenCalais
  • Services like OpenCalais, AlchemyAPI and Zemanta also use Freebase data to look up named entities in text.
  • They all have APIs
  • They all return Freebase IDs.

Interaction Patterns

Traversing a web of entities

  • This is how users interact with your application.
  • Entities allow us to navigate your data according to real world things.
  • Also allows us to describe meaningful relationships between things.

Faceting Pattern

Problem

User wants to find and filter interesting data.

Knowledge Graph is large and not all the data is going to interesting to the user.

No single way to organize data in order to surface interesting results.

Faceting Pattern

Solution

Count up how frequently each facet (property or value) occurs and allow the user to filter the data.

Show different types of facets depending on what data is being shown.
source: Google Search
  • When a user searches for a recipe, Google provides faceted search.
  • The choices for each facet vary depending on the results.
  • Clicking a facet automatically does a new, constrained search.

from apiclient import model

API_KEY = 'YOUR-API-KEY-GOES-HERE'

query = [{'id':None,'name':None,'type':'/people/person','profession':'software developer'}]

for topic in results:
  print topic['name']

Recommendation Pattern

Problem

We want to encourage deeper exploration of our content.

Recommendation Pattern

Solution

Use semantic relationships between entities to make meaningful recommendations for related entities.

Not only can we say that they're related by we can also say why.

  • Entities must be reconciled for this to work.
  • The Knowledge Graph uses Recommendation

Recommendation Pattern

When to use it?

When there are too many possible ways to organize the content and each different users will have varied tastes.
source: Ookaboo
  • Ookaboo is a repository of freely licensed images organized by entity.
  • Every entity has its own page with a description and pictures.
  • The right-hand side has suggestions for other related entities.
  • Ookaboo uses Freebase data to figure out which entities might be related.
  • Makes it much easier to explore a large collection of images.

Presentation Patterns

Mashing up entities on the web

  • The final part of your application.
  • Not just a shiny distraction. Its all based on real data so it adds a lot of value.
  • The key is to mix the KG content with your existing content.

Summarization Pattern

Problem

We want to describe an entity.

User needs clues to know which entity we're talking about.

  • Users don't disambiguate entities with IDs
  • how do they know which "John Smith" we're talking about?

Summarization Pattern

Solution

Display a short summary including name, image, description and relevant property values.

Freebase Topic API

  • One HTTP request gets you everything
  • Gives image, description and all properties
  • Values are formatted so that its easy to display them in HTML

Summarization Pattern

When to use it?

When we have content that needs to be put in the context of one or more entities.

When we want to encourage people to navigate content by entities, we can reinforce that by showing which entities are relevant to each piece of content.
source: The Wall Street Journal
  • WSJ uses Freebase topic summaries to augment movie reviews
  • This article reviews several movies. Each one has a topic on the right.
  • Reconcilation pattern makes this all possible.
  • Also great example of Mashup Pattern. Wouldn't be possible without recon.

Mashup Pattern

Problem

User wants to see different types of information about the same entity.

  • Different sites or data sources.
  • Different types of data about the same entity.

Hard to combine data from different sources.

Mashup Pattern

Solution

Create links between the same entity in different datasets/contexts.

Encourage users to jump between different contexts for the same entity.

  • You can think of these links as "foreign keys" in your database.
source: seevl
  • Seevl is a music recommendation service.
  • It works as a Chrome extension that injects addition information into YouTube pages.
  • When Seevl detects an artist that it knows about, it inserts a topic summary.
  • Also recommends similar artists that you might like.
  • Makes recommendations based on proximity in the knowledge graph and also weights some relationships more than others.

Get all the foreign keys

Freebase API

		import json
		import urllib

		API_KEY = 'YOUR-API-KEY-GOES-HERE'
		service_url = 'https://www.googleapis.com/freebase/v1/topic'
		params = {
		  'filter': ['/type/object/key','/common/topic/topic_equivalent_webpage'],
		  'alldata': true,
		  'key': API_KEY
		}
		topic_id = '/en/san_francisco'
		url = service_url + topic_id + '?' + urllib.urlencode(params)
		response = json.loads(urllib.urlopen(url).read())
		for result in response['result']['property']['/common/topic/topic_equivalent_webpage']['values']
		  print result['uri']
  • Topic API: New API that we're launching
  • Lets you get all the property values for an entity.
  • HIGHLIGHT: We've constrained it to just /type/object/key and /common/topic/topic_equivalent_webpage -> foreign keys

Mashup Pattern

When to use it?

Need to see information from different perspectives.

Different information about different entities is not available from the same source or website.
source: SameAs.org
  • Sameas.org is a data dub that links together different datasets on the web.
  • It has a simple JSON API that you can use.
  • Every entity has a cluster of IDs which are all said to be the "same as" each other.
  • This lets you easily combine data from datasets.

Visualization Pattern

Problem

We want to display many different entities.

Showing summaries for each entity is repetitive and doesn't fully show the relationships between the entities.

Visualization Pattern

Solution

Use data visualization techniques to display many different entities at once in a semantically meaningful way.

  • Trees
  • Maps
  • Timelines
  • Graphs
  • Bubble Charts

  • trees - families or organizations
  • maps - places or events
  • timelines - events
  • graphs - relationships
  • bubble charts - multiple variables
source: Thinkbase

Visualization Pattern

When to use it?

When its important to be able to see the relationships between multiple entities at the same time.
source: conflicthistory.com
  • Conflict history uses Freebase data to show an interactive history of military conflicts
  • Uses both maps and timeline visualizations.
  • Also shows summaries - great example of presentation
